# Краткий гид по структуре папки frontend

- **src/pages/** — страницы приложения (Login, Register, Home, Profile, Hotels). Каждая страница — отдельный экран.
- **src/services/** — сервисы для работы с API (authService.ts — авторизация, hotelService.ts — отели и т.д.). Здесь логика запросов к серверу.
- **src/utils/** — вспомогательные функции (например, работа с токенами в auth.ts).
- **src/App.tsx** — главный компонент приложения, здесь настраиваются маршруты (роутинг) и навигация.
- **src/App.css** — основные стили для всего приложения.
- **public/** — статические файлы (иконки, favicon и т.д.).
- **index.html** — шаблон главной страницы (точка входа для фронта).
- **vite.config.ts** — настройки сборки и proxy для API.
- **package.json** — зависимости и npm-скрипты (запуск, сборка, установка пакетов).

**Взаимосвязи:**
- Страницы используют сервисы для запросов к API.
- App.tsx управляет переходами между страницами.
- Стили из App.css применяются ко всем компонентам.
- Все запросы к /api/* автоматически проксируются на backend (порт см. vite.config.ts).

**Для запуска:**
1. Установить зависимости: `npm install`
2. Запустить: `npm run dev`

**Если нужно добавить новую страницу:**
- Создай файл в src/pages, добавь маршрут в App.tsx и, если нужно, сервис в src/services.
